# design index

Catalogue of every entry reachable from the `/design` router, sourced live
from the `open-design` submodule (`repos/open-design/`). Four axes:

- **skills/** — functional capabilities (briefs, audits, critiques, asset work)
- **design-templates/** — rendering shapes (landing pages, decks, dashboards, posters…)
- **design-systems/** — brand visual languages (a 9-section `DESIGN.md` per brand)
- **craft/** — universal craft rules layered on top (typography, color, a11y, anti-AI-slop…)

A typical build picks **one** design-template (the shape) + **one** design-system
(the look) + the relevant **craft** sections (the rules). Functional **skills** are
invoked mid-task. See `SKILL.md` for the routing procedure.
